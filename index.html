99.99% of the world's data is too sensitive to share. OpenMined builds technology for secure computation across siloed data, preserving attribution-based control. PySyft changes that.

Framework

Structured Transparency

Five pillars for reasoning about information flows. Click each to learn more.

🔐

Input Privacy

Protect the data going in

In our traffic demo, drivers' raw GPS coordinates never leave their device. PySyft uses secure computation so the analysis function executes where the data lives, not in a central server. Technologies: Secure Multi-Party Computation, Homomorphic Encryption.
🔬

Output Privacy

Protect what comes out

The researcher only receives the aggregate result ("HIGH congestion" or "FREE FLOW"), never individual coordinates. Differential privacy adds calibrated noise to prevent anyone from inferring a single driver's location. Technologies: Differential Privacy (via OpenDP).
✅

Input Verification

Verify data authenticity

How do we know the GPS data is real and not fabricated? Input verification ensures data provenance. Records must come from genuine sources. In PySyft, data owners vouch for their datasets when they load them into a Datasite. Technologies: Zero-Knowledge Proofs.
📊

Output Verification

Verify computation correctness

The data owner needs assurance that the researcher's code computes a traffic density metric and nothing else. PySyft's code review workflow lets data owners inspect and approve the exact computation before it runs.
🚧

Flow Governance

Control who can do what

Attribution-Based Control: data owners decide which computations their data supports, and can revoke access at any time. Every computation request must be explicitly approved. This is the foundation of a sustainable data economy.

Developer Quickstart

Build this in 5 steps

Real PySyft API calls. Copy, paste, and run.

# Step 1: Data Owner launches a Datasite import syft as sy # Start a local Datasite server server = sy.orchestra.launch( name="nyc-traffic-datasite", port=8080, dev_mode=True ) # Login as the data owner owner_client = server.login( email="owner@traffic.org", password="changeme" )
# Step 2: Load private GPS data with mock for researchers import numpy as np # Real private data (only the owner sees this) private_coords = np.array([ [40.7128, -74.0060], # Driver 1 [40.7131, -74.0058], # Driver 2 [40.7129, -74.0062], # ... 15 records total ]) # Mock data (researchers prototype against this) mock_coords = np.random.uniform( low=[40.70, -74.02], high=[40.73, -74.00], size=(15, 2) ) # Upload as a Syft Dataset dataset = sy.Dataset( name="NYC GPS Records", asset_list=[ sy.Asset( name="coordinates", data=private_coords, mock=mock_coords, ) ] ) owner_client.upload_dataset(dataset)
# Step 3: Researcher connects and submits analysis researcher_client = sy.login( url="http://localhost:8080", email="researcher@university.edu", password="changeme" ) # Define the privacy-preserving computation @sy.syft_function_single_use(coords=coords_asset) def analyze_traffic(coords): """Compute traffic density from GPS spread.""" lat_std = coords[:, 0].std() if lat_std < 0.005: return "HIGH CONGESTION" return "FREE FLOW" # Submit request (data owner must approve) researcher_client.code.request_code_execution(analyze_traffic)
# Step 4: Data Owner reviews and approves # Owner sees the pending request requests = owner_client.requests request = requests[0] # Inspect the code (manual review step) print(request.code) # Output: def analyze_traffic(coords): # lat_std = coords[:, 0].std() # if lat_std < 0.005: # return "HIGH CONGESTION" # return "FREE FLOW" # Approve -- code will now run on REAL private data request.approve()
# Step 5: Researcher retrieves the result result = researcher_client.code.analyze_traffic( coords=coords_asset ) print(result.get()) # Output: "HIGH CONGESTION" # The researcher got the traffic insight # but NEVER saw a single GPS coordinate. # The data never left the Datasite. # Privacy preserved.
